# Инструкция по работе с функционалом ROP Role

> **Для кого:** РОП, менеджер, администратор

## 1. Что входит в функционал

Функционал включает:

- работу РОПа с командой менеджеров и клиентами;
- доступ к заказам в своей зоне ответственности;
- начисления менеджеров и РОПа;
- аналитику начислений для контроля результатов.

Баланс и каталоги описаны в отдельных инструкциях.

## 2. Роли и зоны ответственности

- **Администратор (1,2):** назначает менеджеров РОПам, видит все команды, настраивает проценты, смотрит общую аналитику.
- **РОП (6):** работает со своей командой менеджеров и закрепленными клиентами, видит начисления команды.
- **Менеджер (5):** работает только со своими клиентами и видит свои начисления.

## 3. Базовый сценарий запуска

1. Администратор назначает пользователю роль РОП.
2. Администратор формирует команду РОПа (назначает менеджеров).
3. РОП распределяет клиентов между менеджерами команды или берет клиента на себя.
4. РОП/менеджер работают с заказами в своей зоне ответственности.
5. Администратор настраивает процентные ставки.
6. При переводе заказа в статус `Готов` создаются начисления.

## 4. Работа с командой РОПа

### Для РОПа

- Открыть пункт меню **Команда РОПа**.
- Просмотреть менеджеров своей команды.
- Назначить клиента менеджеру команды.
- Переназначить клиента другому менеджеру или себе.

Важно:

- РОП не может назначать клиентов менеджерам вне своей команды.
- Состав команды РОП не изменяет, это делает администратор.

### Для администратора

- Открыть пункт меню **Команды РОПов**.
- Выбрать карточку РОПа и перейти в клиентов команды.
- Управлять закреплениями на экране команды выбранного РОПа (режим администратора).

## 5. Фильтрация заказов

- По умолчанию РОП и менеджер видят только свою зону ответственности.
- Для временного полного просмотра доступен режим `scope=all` (переключатель «Все заказы»).
- Состояние `scope=all` не сохраняется между сессиями.

## 6. Начисления и проценты

### Настройка процентов (администратор)

- Менеджеру задается процент по типам изделий.
- РОПу задаются два типа процентов:
    - `own` — по своим клиентам;
    - `manager` — по клиентам менеджеров своей команды.

### Начисления

- Триггер начисления: статус заказа `Готов`.
- База расчета: полная сумма заказа.
- Старые начисления не пересчитываются при изменении ставок.

## 7. Где смотреть начисления

- **РОП:** пункт меню **Начисления** + блок начислений менеджеров команды.
- **Менеджер/РОП:** счетчик в шапке (сумма за текущий месяц).
- **Администратор:** пункты меню **Аналитика начислений** и **Команды РОПов**.
